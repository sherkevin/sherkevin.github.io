# Homepage Writing Rules

This file records the writing style requirements for `sherkevin.github.io`.

## Core Style

1. Use a calm, objective, restrained tone.
2. Do not write from a recruiting perspective.
3. Do not explicitly praise the person.
4. Do not explain traits directly. Let facts imply traits.
5. Do not write sentences such as "形成了 xxx 能力/习惯", "体现了 xxx", "让我意识到 xxx" in timeline or experience sections.
6. Timeline and experience sections should describe what happened at that time: institution, project, task, method, output, award, result.
7. Reflection and abstractions belong only in `技术观` or `方法论`, not in event descriptions.
8. The tone should be factual, slightly distant, and self-contained: not written to persuade recruiters, but to document a technical person.

## Preferred Writing

Use:

- `就读于河南睢县高级中学；参加高中物理竞赛，获省级二等奖`
- `参与 MCP Factory 流水线建设，将内部 API 转化为标准化 Tool Call`
- `围绕 RAG 检索、引用证据、SOP 安全与人工升级设计诊断流程`

Avoid:

- `形成理工科基础和长期自驱学习习惯`
- `这段经历让我意识到...`
- `我选择...是因为...`
- `这个项目体现了我的...`
- `为了求职/投递/岗位匹配...`

## Check Before Publishing

Search for and remove self-promotional or retrospective wording:

- `让我`
- `我意识`
- `我选择`
- `形成`
- `体现`
- `求职`
- `候选人`
- `岗位`
- `招聘`
- `offer`
