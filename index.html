关于

我长期关心一个问题：智能系统如何在真实世界里可靠地工作。这里的“可靠”包括多模态模型是否有视觉证据， RAG 回答是否有可追溯引用，推荐系统的多模态增量是否真实，Tool Use 是否有清晰 schema 和错误恢复，复杂 Agent 工作流是否可控。

我的经历由几条线索交织而成：竞赛训练带来的算法基本功，本科和清华阶段的 AI 学习路径， MiniMax 的多模态推荐与 LLM4Rec，字节的 Agent Infra，SMEE 的领域 LLM/RAG 工业诊断， 以及甲己无形、AgentOS、Weaver 等应用型智能体项目。

技术观

智能是压缩，更高级的智能是重建

压缩、解压缩、重建都是智能形式。压缩让世界变得可表示，解压缩让表示重新展开， 而重建要求系统把碎片还原为结构；我认为重建是对世界最深层次的理解。

Memory、Tools 与 Environment 是三种不同东西

LLM + memory 更像大脑，tools 更像身体，environment 才是世界。 Memory 保存反思和压缩过的经验；environment 保存代码、文档、数据和协作空间中的真实产出。

Attention 是人机协同的核心

构建上下文时，人类用注意力圈定什么重要、什么该被暂时忽略。 这也是全自动流程和人机协同的本质区别：上下文不是越多越好，而是要被正确选择。

频率塑造记忆，价值驱动反思

模型学习的是世界的普遍频率，个人记忆则来自每个人经历过的特异频率。 价值高的事件会触发反思，反思相当于对高价值样本进行迭代压缩和再训练。

AI Coding 的关键是熵控与最小改动

代码是智能体沉淀到 environment 中的产出。好的 AI coding 不是大面积生成， 而是在理解项目结构后做最小必要修改，控制代码熵，让系统继续保持可读、可维护、可回滚。

方法论

  • 先定义 failure mode，再选择模型或系统方案。
  • 先设计公平评测，再讨论方法是否有效。
  • 复杂方法必须有放弃条件，不能只因 fancy 而上线。
  • 大模型应用的核心不只是生成结果，而是证据、约束、反馈和可控性。
